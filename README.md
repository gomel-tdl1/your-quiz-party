# Своя игра

Квиз-игра в формате «Своя игра» / Jeopardy для вечеринок: поле из категорий и
вопросов по стоимости, полноэкранный показ вопроса на проекторе и кнопки для
начисления баллов командам. Только фронтенд (React + Vite + TypeScript) + Supabase
для хранения. Бэкенда нет — фронт ходит в базу напрямую.

## Что умеет

- **Поле** — категории-колонки × стоимости-строки. Клик по ячейке разворачивает
  вопрос на весь экран эффектом «захвата» (плитка вырастает в карточку).
- **HTML-вопросы** — тело вопроса и ответа хранят HTML и рендерятся через
  `dangerouslySetInnerHTML` (после чистки DOMPurify). Можно вставлять `<img>`,
  `<audio>`, `<video>`, `<iframe>` (YouTube), форматирование.
- **Счёт** — табло команд. Внутри вопроса ведущий жмёт на команду, чтобы начислить
  (или снять) баллы. Есть ручная корректировка ±100.
- **Редактор** — категории, ячейки, стоимости, команды; живое превью HTML.
- **Экспорт/импорт JSON** — делиться готовыми квизами.
- **Новая партия** — открыть все ячейки заново и обнулить счёт.

## Запуск

```bash
npm install
```

1. Создай бесплатный проект на [supabase.com](https://supabase.com).
2. **Project Settings → API** → скопируй `Project URL` и `anon public` ключ.
3. **Settings → Database → Connection string → URI** → скопируй строку (подставь пароль базы).
4. Создай `.env.local` в корне:

   ```env
   VITE_SUPABASE_URL=https://xxxx.supabase.co
   VITE_SUPABASE_ANON_KEY=eyJhbGciOi...
   SUPABASE_DB_URL=postgresql://postgres:ПАРОЛЬ@db.xxxx.supabase.co:5432/postgres
   ```

5. Запусти:

   ```bash
   npm run dev
   ```

Таблицы создаются **автоматически**: перед стартом дев-сервера прогоняется
`scripts/migrate.mjs`, который накатывает [`supabase/schema.sql`](supabase/schema.sql)
по прямому подключению к БД (`SUPABASE_DB_URL`). Скрипт идемпотентный — гоняется
при каждом старте без вреда. Накатить вручную в любой момент: `npm run migrate`.

При первом запуске сам создаётся демо-квиз «Викторина на вечер» с примерами
картинки и аудио — можно сразу играть, потом переписать под себя в редакторе.

> Почему отдельная строка подключения? `anon`-ключ ходит через PostgREST, который
> не умеет выполнять `CREATE TABLE`. Поэтому миграция идёт напрямую в Postgres.
> `SUPABASE_DB_URL` используется **только** локальным скриптом и не попадает во фронтенд.

## Как играть на вечеринке

1. Подключи ноутбук к проектору, открой приложение.
2. В **Редакторе** заведи команды и впиши свои вопросы (или импортируй JSON).
3. Жми **← Играть**. Кликай ячейки, читай вопрос, жми **Показать ответ**.
4. Кто ответил — нажми на команду, баллы начислятся. **Готово ✓** закрывает ячейку.

## Заметки

- Доступ открыт для `anon`-ключа (см. политики в `schema.sql`) — это нормально
  для приватной игры. Для публичного хостинга добавь авторизацию и ужесточи RLS.
- HTML вопросов пишешь ты сам; DOMPurify страхует от случайно вставленного
  опасного кода, но всё равно вставляй медиа из источников, которым доверяешь.
